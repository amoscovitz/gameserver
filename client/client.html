This is Arnaud Moscovitz client/server sample project
I did a simple server game oriented in C++ based on the book "Coding complete 4th edition". That includes Non blocking I/O and a Game event manager.I have added the HTTP communication and some design patterns just for showing
As a client I wanted to try the basics for a game in HTML5 (PC and mobile). So here it is. Based on the website of jlongster
There is plenty of improvements to make either on client or server that I am aware of. But as you know, you have to deliver something at a time. That is why I insist on the word :"sample".
Few things in my mind:
-Client: Make it in Unity! And why not a multiplayer approach (Bomberman like)
-Server: Multiplayer mode (Concurrency dev approach).
The code needs some const-ness and magic numbers reinforcement. I did not manage yet requests order in the server.
Client has light event management

Add timeout parameter in url to change ajax request timeout i.e : client.html?timeout=5000 for 5secs
Browser console is used for messages output

Actual architecture:
(Musimos.com)HTML5 Ajax with jquery HTTPRequest when click ====> (Personal computer) HttpRequest C++ server
